Comandos Git



git Version


# Mostra a versão instalada do git.
$ git version


#Cria um arquivo "README.md" com o conteudo "# git_command"
$ echo "# git_command" >> README.md

git init


# Cria um repositorio vazio dentro da pasta que estiver posicionado.
$ git init

git status


# Mostra as diferenças entre diretorio do projeto (Diretorio de trabalho) e a area de staging (conteiner do git).
$ git status

#Fluxo de trabalho
#O Git Possui 3 repositórios ("árvores") locais mantidas pelo git
1# "Working Directory" é a areas que contém os arquivos vigentes.
2# "Index" (area Staging),funciona como uma área temporária (staging)
3# "HEAD" area que aponta para o último commit (confirmação das alterações).

git add


# git add - inclui alterações dos arquivos da area de trabalho para a area de stanging.
$ git add

#inclui todos arquivos alterações dos arquivos da area de trabalho para a area de stanging.
$ git add -A
# Este comando adiciona o(s) arquivo(s) em um lugar que chamamos de INDEX(Area staging), esta area armazena os arquivos para posteriormente serem enviados para o repositório do github.
# É importante: O comando ADD não adiciona automaticamente o(s) arquivo(s) ao repositório, apenas prepara para entrar na próxima revisão do repositório.(O envio é feito ao executar o comando push, ver abaixo)


git config


# Configurando o git
$ git config --global user.email "datacrusher@outlook.com"
$ git config --global user.name "datacrusher"
# Estas configurações ficam alocadas no arquivo ~/.gitconfig, onde o ~ é o seu diretório home.
# No Windows, ele fica em c:\Usuarios\\.gitconfig.

git commit


#Confirma (salva) as mudanças (alterações) feitas anteriormente.
$ git commit - m 'Primeiro Commit'

git log


#Visualizar o Histórico de Commits
$ git log

#--oneline
#Visualizar o Histórico de Commits. cada commit sera mostrado em uma linhas unica
$ git log --oneline

#--decorate
# Visualizar o Histórico de Commits. cada commit sera mostrado em uma linhas unica e mostra as linhas do tempo (branchs atual)
$ git log --oneline --decorate

#-all
# Visualizar o Histórico de Commits. cada commit sera mostrado em uma linhas unica e mostra as linhas do tempo (todos os branchs)
$ git log --oneline --decorate --all

#-graph
# Visualizar o Histórico de Commits. cada commit sera mostrado em uma linhas unica e mostra as linhas do tempo (todos os branchs)
$ git log --oneline --decorate --all --graph


# Criando um alias para comando log
$ git config --global alias.logall "log --oneline --decorate --all --graph"
$ git logall

#Para subir para o github:
# define o diretorio(projeto) do github que será atualizado.
$ git remote add origin https://github.com/Datacrusher/git_command.git

#Push (empurrar) é usado para publicar todos os seus commits para o github. Neste momento, será pedido a sua senha.
$ git push -u origin master

Branch



#Branch é uma nova linha de desenvolvimento (linha de tempo)independente da principal (master). [traduções: Seção, ramo, galho , subdividir-se ]


git branch


#Visualizar as linhas do tempo criadas.(digite o comando abaixo)
$git branch

#Criando uma nova linha de tempo (branch)
$ git branch
# Ex:
$ git branch line1

# O comando Cat serve para criar ou mostrar o conteúdo de um arquivo.
# no exemplo abaixo o conteudo mostra o ponteiro para o ultimo commit das Branchs
$ cat .git/refs/heads/master
$ cat .git/refs/heads/line1


# Exclui uma linha do tempo(branch)
$ git branch -d line1



# HEAD (area de trabalho staging)
# mosca o local atual de trabalho (staging)
$ cat .git/head

git checkout


# mudar de Branch (linha de tempo)
# o comando abaixo Move o Head do branch atual (master) para o (line1) criado anteriormente.
$ git checkout line1

# verificando o head (area de trabalho) atual.
$ cat .git/head

#Cria uma nova Branch e aponta (torna) o novo Branch como area atual de trabalho
$ git checkout -b line2





Merging





git merge


#Merge -

# muda a area de trabalho (branch) para Master
$ git checkout master
# Atualiza as mudanças da branch "line1" para a branch "master"
$git merge line1

#tipos
#"Fast-forward" é atualização de referência, não tendo conflitos (divergencias)
#"Recursive Strategy"
